# Design Map

## Spacing Scale

- 3.2px：页脚并列链接的右间距
- 16px：正文段落的垂直 margin，也是重复次数最多的间距
- 24px：链接列表的行高与行步进
- 28.8px：信息区顶部 padding
- 36px：主视觉区底部 padding
- 72px：信息区底部 padding
- 144px：桌面左右外边距与主视觉顶部 padding

## Font Hierarchy

- Hero：115.2px / 500 / 149.76px，`nbintl`，字距 -1.728px
- Body：16px / 400 / 20.8px，`nbintl-mono`，字距 0.8px
- Link list：16px / 400 / 24px，`nbintl-mono`，字距 0.8px
- 可见字号比：115.2:16 = 7.2:1

## Color Palette

- 页面背景：`#FFFFFF`
- 主文字与链接：`#111111`
- Hero 链接下划线：`rgba(0, 0, 0, 0.20)`，4px
- 正文链接下划线：`rgba(17, 17, 17, 0.40)`，1px

## Image Ratios

- 内容图片：无
- 卡片媒体：无

## Component Tokens

- 内容宽度：1152px（1440px 视口）
- 外边距：左右各 144px
- 下方布局：2 列 × 576px，gutter 0px
- 圆角：0px
- 站点级阴影：无
- 卡片：无
- 主要对齐：左对齐，起始轴为 x=144px 与 x=720px
- 动效：header 2s、信息区 1.2s、状态文本 0.8s；链接使用 0.65s `cubic-bezier(0.16, 1, 0.3, 1)`

# Taste DNA

### 留白就是分区

- **Trigger**: 当首句与个人信息需要被区分时
- **Decision**: 选择 630px 高的首屏和连续白底，而不是分割线、背景色块或卡片
- **Reason**: 读者会先停在唯一的大句子上，再自然下移到事实信息；分区不需要额外图形解释
- **Evidence**: header 高 630px；主句仅占约 150px 高；卡片、圆角和站点级阴影数量均为 0

### 两级字号，拒绝中间态

- **Trigger**: 当一句身份声明需要压过其余所有信息时
- **Decision**: 选择 115.2px 与 16px 的 7.2:1 跳差，而不是建立多档标题字号
- **Reason**: 读者不会犹豫哪里是主角，次要内容则保持相同语气供快速浏览
- **Evidence**: 可见字号仅 115.2px 和 16px；字重仅 500 和 400；主句行高 149.76px，正文行高 20.8px 或 24px

### 单轴骨架

- **Trigger**: 当 1440px 画布只有少量文字需要组织时
- **Decision**: 选择 x=144px 与 x=720px 两条固定起始轴，而不是居中卡片或分散对齐
- **Reason**: 可预测的起点让大面积空白仍有秩序，眼睛不必反复寻找下一段
- **Evidence**: 左右外边距各 144px；内容宽 1152px；下方两列各 576px 且 gutter 为 0px

### 动作只证明页面活着

- **Trigger**: 当静态个人主页需要一点持续存在感时
- **Decision**: 选择实时年龄数字与克制的进入动效，而不是背景动画、插画或滚动特效
- **Reason**: 变化传达的是人的时间正在流动，不会从姓名、工作和链接上夺走注意力
- **Evidence**: 年龄保留 20 位小数并持续更新；进入动画为 2s、1.2s 和 0.8s；页面没有内容图片，主链接只使用 4px 半透明下划线
